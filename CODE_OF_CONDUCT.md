# Code of Conduct

## Our Commitment

The Eagle community is committed to providing a welcoming, inclusive, and respectful environment for all contributors. We value collaboration, innovation, and responsible AI development.

As an open-source, production-grade surveillance AI project, we encourage technical discussions, constructive feedback, and ethical contributions that advance the project's goals.

---

## Expected Behavior

All contributors are expected to:

* Be respectful and professional in discussions.
* Provide constructive feedback and code reviews.
* Collaborate openly and respectfully with other contributors.
* Follow project guidelines and maintain code quality.
* Respect differing viewpoints and experiences.
* Contribute responsibly to AI and computer vision technologies.
* Help foster a positive and inclusive community.

---

## Unacceptable Behavior

The following behaviors are prohibited:

* Harassment, discrimination, or hateful conduct.
* Personal attacks, insults, or abusive language.
* Trolling, spamming, or intentionally disruptive behavior.
* Sharing private or confidential information without permission.
* Submitting plagiarized work or misrepresenting authorship.
* Promoting unethical, illegal, or harmful uses of the project.

---

## Responsible AI Development

Contributors should:

* Develop features responsibly and ethically.
* Consider privacy, security, and fairness when proposing changes.
* Report vulnerabilities responsibly.
* Avoid contributions intended to enable unlawful or harmful activities.

---

## Enforcement

Project maintainers have the responsibility to review reported incidents and take appropriate action when this Code of Conduct is violated.

Possible actions include:

1. Warning the contributor.
2. Requesting corrective action.
3. Restricting repository participation.
4. Removing contributions.
5. Permanent removal from the community for severe violations.

---

## Reporting

If you experience or witness behavior that violates this Code of Conduct, please report it to the project maintainers through GitHub Issues, Discussions, or other designated communication channels.

All reports will be reviewed fairly and handled appropriately.
